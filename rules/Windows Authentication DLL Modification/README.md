# Windows Authentication DLL Modification

## Overview

This detection rule monitors changes to **cryptdll.dll** and **samsrv.dll** within the protected Windows system directories **C:\Windows\System32** and **C:\Windows\SysWOW64**.

These DLLs are essential components of the Windows authentication and cryptographic infrastructure. Unauthorized creation, modification, renaming, or deletion of either DLL is uncommon and may indicate attempts to modify the Windows authentication process, establish persistence, bypass security controls, or prepare the system for credential theft.

Because these files are rarely modified outside of Windows servicing operations, alerts generated by this rule should be treated as high priority.

## Detection Logic

The rule performs the following actions:

1. Monitors changes to:
   - `cryptdll.dll`
   - `samsrv.dll`
2. Restricts monitoring to:
   - `C:\Windows\System32\`
   - `C:\Windows\SysWOW64\`
3. Detects the following actions:
   - File creation
   - File modification
   - File renaming
   - File deletion
4. Returns the initiating process and associated file metadata for investigation.

## Investigation Guidance

When this rule generates an alert:

1. Verify whether the activity occurred during a Windows Update or approved maintenance window.
2. Validate the digital signature of the modified DLL.
3. Compare the file hash against known-good Microsoft versions.
4. Review the initiating process and parent process.
5. Investigate any registry modifications affecting Windows authentication.
6. Review LSASS activity, credential access alerts, and service creation events.
7. Determine whether additional persistence mechanisms were established.
8. Isolate the endpoint if unauthorized modification is confirmed.

---

## False Positives

Potential legitimate triggers include:

- Windows Update
- Component-Based Servicing (CBS)
- Deployment Image Servicing and Management (DISM)
- System File Checker (SFC)
- Microsoft servicing operations
- Approved software deployment tools

---

## Tuning Recommendations

- Exclude trusted Windows servicing processes after validation.
- Correlate with:
  - DeviceRegistryEvents
  - DeviceProcessEvents
  - DeviceImageLoadEvents
  - LSASS access detections
  - Service installation events
- Prioritize:
  - Unsigned DLLs
  - Non-Microsoft initiating processes
  - Modifications performed outside maintenance windows

---

## Limitations

This rule monitors only `cryptdll.dll` and `samsrv.dll`. Other Windows authentication components or protected system binaries are not included and should be monitored through additional integrity monitoring rules.
